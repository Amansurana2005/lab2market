## Quick Deploy to Render (Easiest)

1. **Create MongoDB Atlas Account**
   - Go to [MongoDB Atlas](https://www.mongodb.com/cloud/atlas)
   - Create a free cluster
   - Copy your connection string (looks like: `mongodb+srv://username:password@cluster.mongodb.net/lab2market`)

2. **Push to GitHub**
   - Push this entire repo to GitHub (including `render.yaml`)

3. **Deploy on Render**
   - Go to [Render.com](https://render.com) and sign up
   - Click **"New +"** â†’ **"Blueprint"**
   - Connect your GitHub repo
   - Render will auto-detect `render.yaml` and create both services
   - Fill in these environment variables:
     - **Backend Service** (`lab2market-backend`):
       - `MONGO_URI` = your MongoDB connection string
       - `JWT_SECRET` = any random string (e.g., `super-secret-key-12345`)
       - `FRONTEND_URL` = your deployed frontend URL (will be auto-generated by Render)
     - **Frontend Service** (`lab2market-frontend`):
       - `VITE_API_URL` = will be `https://lab2market-backend.onrender.com/api`

4. **Wait for deployment**
   - Both services will build and deploy automatically
   - Frontend will be live at: `https://lab2market-frontend.onrender.com`
   - Backend will be live at: `https://lab2market-backend.onrender.com`

## Local Development

```bash
# Terminal 1 - Backend
cd backend
npm install
npm start

# Terminal 2 - Frontend
cd frontend
npm install
npm run dev
```

Visit `http://localhost:5173` in your browser.

## Manual Deployment (if render.yaml doesn't work)

**Backend (Web Service)**
- Root Directory: `backend`
- Build Command: `npm install`
- Start Command: `npm start`
- Health Check Path: `/api/health`
- Environment Variables:
  - `MONGO_URI`
  - `JWT_SECRET`
  - `FRONTEND_URL`

**Frontend (Web Service)**
- Root Directory: `frontend`
- Build Command: `npm install && npm run build`
- Start Command: `npm run preview`
- Environment Variables:
  - `VITE_API_URL` = backend URL + `/api`
